 To verify Fuseki's data persistence:

1. Check the volume mount in docker-compose.yml:
```yaml
volumes:
  - ./fuseki-data:/fuseki
  - ./config.ttl:/fuseki/config.ttl
```

2. Verify fuseki-data directory contains:
- databases/
- backups/
- shiro.ini
- config.ttl

3. Test data persistence:
```bash
# Add test data
curl -X POST http://localhost:4030/ds/update \
  -H "Authorization: Basic $(echo -n 'admin:admin123' | base64)" \
  -H "Content-Type: application/sparql-update" \
  --data "INSERT DATA { <http://example/s> <http://example/p> <http://example/o> }"

# Restart containers
docker-compose restart

# Query to verify data remains
curl -X GET http://localhost:4030/ds/query \
  -H "Accept: application/sparql-results+json" \
  --data-urlencode "query=SELECT * WHERE { ?s ?p ?o }"
```
