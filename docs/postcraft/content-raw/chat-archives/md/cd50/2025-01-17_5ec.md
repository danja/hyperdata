 You're right - the error is in npm itself, not the project. Looking at the error message:

"Exit prior to config file resolving" + "call config.load() before reading values"

This indicates npm's own configuration system is failing. Check:
```bash
which npm
ls -l $(which npm)
```

This will show if npm's installation is corrupted or symlinked incorrectly.

q1: What's the output of `which npm`?
q2: Does `nvm use system` change behavior?
q3: Does running as root show same error?
q4: Is npm's global config readable?
